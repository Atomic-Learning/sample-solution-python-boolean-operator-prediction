Case A: not True

The not operator reverses the boolean value, so not True returns False.

print(not True)

Case B: or with comparison and in

This evaluates to (3 > 4) or (1 in [1, 2]). The first part is False (3 is not greater than 4), and the second part is True (1 is in the list). Since we have False or True, the result is True.

print(3 > 4 or 1 in [1, 2])

Case C: Modulo and not-equal with and

This evaluates to (10 % 2 == 0) and (5 != 6). The modulo 10 % 2 equals 0 (10 is even), so the first part is True. The second part, 5 != 6, is also True. Since both are true, and returns True.

print(10 % 2 == 0 and 5 != 6)

Case D: Multiplication, comparison, indexing, and and

This evaluates to (5 * 2 > 8) and (list1[1] == 1). The first part, 5 * 2 > 8, is 10 > 8, which is True. The second part, list1[1], accesses the second element of the list (which is 2), so we have 2 == 1, which is False. Since we have True and False, the result is False.

list1 = [1, 2] print(5 * 2 > 8 and list1[1] == 1)

Case E: String membership and modulo with or

This evaluates to ("world" in string1) or (10 % 3 == 0). The first part, "world" in string1, checks if the substring "world" is in string1. Since string1 is "Hello World" (with a capital W), and Python is case-sensitive, "world" is not found, so this part is False. The second part, 10 % 3 == 0, checks if the remainder when 10 is divided by 3 is 0. 10 % 3 is 1, so this is also False. Since both sides are False, the result is False.

string1 = "Hello World" print("world" in string1 or 10 % 3 == 0)
